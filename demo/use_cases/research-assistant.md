# Use Case: Research Assistant for Product Strategy

## Scenario
A product manager needs a structured briefing on AI meeting assistants for enterprise customers.

## Reproducible Input Prompt
```text
Create a 1-page strategy brief comparing AI meeting assistants for enterprise.

Deliver:
1) competitor matrix (top 5 vendors),
2) buyer priorities by persona (CIO, IT admin, end user),
3) key risks (security/compliance/adoption),
4) recommendation for a mid-market SaaS company entering this segment.

Output format:
- bullet points only,
- include assumptions and confidence level,
- end with a 30/60/90-day plan.
```

## Expected Output (Golden)
- Includes 5-vendor matrix with clear dimensions (pricing model, security posture, integrations, differentiator).
- Separates priorities by persona and references enterprise buying constraints.
- Identifies risks: data retention, model hallucination, and user adoption fatigue.
- Ends with realistic 30/60/90 plan with measurable milestones.

## Evaluation Criteria
- **Coverage (35%)**: All requested sections included.
- **Depth (25%)**: Insights go beyond generic statements.
- **Decision quality (25%)**: Recommendation ties to assumptions and risk profile.
- **Formatting compliance (15%)**: Uses requested bullet-only format.
